About VidEdit

A free, open-source video editor that runs entirely in your browser — no uploads, no accounts, no compromise.

What is VidEdit?

VidEdit is a browser-based video editor built on the philosophy that powerful creative tools should be free, accessible, and private. Unlike cloud-based editors that require you to upload your footage to their servers, VidEdit processes everything locally in your browser using modern Web APIs.

Whether you need to quickly trim a video for social media, add captions to a tutorial, adjust colors on a clip, or merge multiple recordings into one — VidEdit handles it all without ever seeing your files.

How It Works

VidEdit uses a suite of browser APIs that have matured significantly in recent years:

  • File API — reads your video from disk into browser memory without uploading
  • HTML5 Video Element — decodes and plays back any browser-supported video format
  • Canvas API — renders frames, applies filters, draws text overlays
  • MediaRecorder API — records the processed canvas output as a video stream
  • Web Audio API — handles volume, muting, and audio processing
  • CSS Filters — applies real-time visual filters during playback preview

Technology Stack

HTML5 CSS3 Vanilla JS Canvas API MediaRecorder API File API Web Audio API localStorage

VidEdit uses zero external JavaScript frameworks or libraries. It runs entirely on native browser APIs, which means it's fast, lightweight, and has no supply-chain dependencies.

Browser Support

VidEdit works best in modern browsers that support the MediaRecorder API:

  • ✅ Google Chrome 60+
  • ✅ Mozilla Firefox 65+
  • ✅ Microsoft Edge 79+
  • ✅ Opera 47+
  • ⚠️ Safari — limited MediaRecorder support; preview works but export may vary
  • ❌ Internet Explorer — not supported

Limitations

Being browser-based comes with some inherent tradeoffs compared to desktop applications:

  • Export performance depends on your device's CPU — very long or high-resolution videos may take time
  • The exported format is primarily WebM (widely supported) with MP4 support where available
  • No multi-track timeline — designed for single-clip editing workflows
  • Safari has limited MediaRecorder support and may not export video

We're continuously working to expand capabilities within the browser sandbox constraints.
